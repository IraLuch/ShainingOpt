
# ShainingOpt — Оптовый интернет-магазин

Проект представляет собой веб-сервис для оптовой торговли (e-commerce), разработанный на платформе **ASP.NET Core**. 

## Ключевой функционал и бизнес-логика

* **Динамический каталог и фильтрация:** Полноценная витрина товаров со сквозной фильтрацией по категориям, брендам, цветам, размерам и диапазону оптовых цен. Реализована кастомная система пагинации (постраничного вывода).
* **Карточка товара и модификации:**  Пользователь может динамически переключаться между модификациями товара (выбор конкретного размера/цвета), при этом на лету обновляются остатки на складе, изображения и минимальное количество для заказа.
* **Гибридная корзина с умным слиянием (Merge Carts):** * Анонимные пользователи могут собирать товары в корзину (идентификация через долгосрочные `Cookies` и `Guid`).
    * При авторизации гостевая корзина автоматически «склеивается» с постоянной корзиной пользователя в базе данных. Количество позиций автоматически суммируется с жестким ограничением по складским остаткам.
* **Интеграция с платежным шлюзом (ЮKassa API):**
    * Формирование уникального номера заказа и инициализация платежной сессии через API ЮKassa.
    * Реализован защищенный эндпоинт (`Webhook`) для приема асинхронных POST-уведомлений от серверов ЮKassa о статусе оплаты.
* **Панель администрирования (Areas):** Разделение интерфейсов и прав доступа с помощью областей (`Admin` / `Manager`) для управления каталогом, заказами, пользователями и т.д.

## Стек технологий

* **Backend:** C#, .NET 10 (ASP.NET Core MVC)
* **Database & ORM:** SQLite, Entity Framework Core 
* **Security:** ASP.NET Core Identity (авторизация по ролям: Admin, Manager, Client), Anti-Forgery Tokens (защита от CSRF-атак)
* **Frontend-логика:** JavaScript (использование Select2), AJAX (Fetch API).
* **Инструменты разработки и тестирования:** Cloudflare Tunnel (`cloudflared`) для локальной отладки входящих вебхуков от платежной системы на `localhost`.

После клонирования репозитория необходимо применить миграции базы данных:

dotnet ef database update

## Локальное развертывание и тестирование Webhooks

Поскольку платежный шлюз ЮKassa не может отправлять уведомления на `localhost`, для локальной отладки и тестирования успешной оплаты использовался daemon-туннель **Cloudflare Tunnel**.

1.  Запустите локальный проект на выбранном порту (например, `https://localhost:7224`).
2.  Запустите туннель командой:
    ```bash
    cloudflared tunnel --url http://localhost:5212
    ```
3.  Полученный временный адрес `*.trycloudflare.com` укажите в личном кабинете ЮKassa в качестве URL для уведомлений:
    `https://your-subdomain.trycloudflare.com/api/payment/webhook`
